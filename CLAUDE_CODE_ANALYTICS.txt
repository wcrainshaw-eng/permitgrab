PART 2: BUILD THE EVENT LOGGING UTILITY
========================================================================================

Create a utility module (e.g., app/analytics.py or utils/analytics.py) with
a single function that every route can call:

    def track_event(event_type, event_data=None, page=None, city_filter=None,
                    trade_filter=None):
        """
        Log a user event. Call this from any route handler.
        Automatically captures: user_id (from session/auth), session_id,
        referrer, user_agent, ip_hash.
        """

Implementation details:
  - Get user_id from the Flask session/login context (None if not logged in)
  - Get or create session_id: check for a 'session_id' cookie. If not present,
    generate a UUID and set it as a cookie (httponly, 30-day expiry). This
    tracks anonymous visitors across page views within a session.
  - Hash the IP: hashlib.sha256(request.remote_addr.encode()).hexdigest()
  - Get referrer from request.referrer
  - Get user_agent from request.headers.get('User-Agent')
  - Insert into user_events table
  - This function should NEVER raise an exception that breaks the page.
    Wrap the entire body in try/except and log errors silently. Analytics
    should never degrade the user experience.

Also create a helper for the session cookie:

    @app.before_request

PART 3: INSTRUMENT ALL ROUTES WITH PAGE VIEW TRACKING
========================================================================================

Add track_event('page_view', page=request.path) to every route handler.

The cleanest way is a @app.after_request decorator or middleware:

    @app.after_request
    def track_page_view(response):
        if response.status_code < 400 and request.endpoint:
            # Don't track static files, API calls, or the health endpoint
            skip_prefixes = ('/static', '/api/', '/health')
            if not any(request.path.startswith(p) for p in skip_prefixes):
                try:
                    track_event(
                        event_type='page_view',
                        page=request.path,
                        event_data={
                            'status_code': response.status_code,
                            'method': request.method
                        }
                    )
                except Exception:
                    pass  # Never break the response
        return response

This gives you automatic page view tracking on every route without touching
individual handlers.

    import time
    import requests
    from datetime import datetime, timedelta

    def check_city_api(city_slug, api_url, api_type):
        """
        Hit the actual data API for a city and verify it returns permits.
        api_type is 'socrata' or 'arcgis' depending on the city's data source.
        """
        start = time.time()
        try:
            resp = requests.get(api_url, timeout=30, params={'$limit': 1} if api_type == 'socrata' else {'resultRecordCount': 1})
            elapsed_ms = int((time.time() - start) * 1000)

            if resp.status_code != 200:
                return {
                    'status': 'error',
                    'response_time_ms': elapsed_ms,
                    'details': {'status_code': resp.status_code, 'body': resp.text[:500]}
                }

            data = resp.json()
            # Socrata returns a list, ArcGIS returns {'features': [...]}
            if api_type == 'socrata':
                record_count = len(data) if isinstance(data, list) else 0
            else:
                record_count = len(data.get('features', []))

            return {
                'status': 'ok' if record_count > 0 else 'degraded',
                'response_time_ms': elapsed_ms,
                'details': {
                    'record_count': record_count,
                    'api_type': api_type,
                    'status_code': 200
                }
            }
        except requests.Timeout:
            return {'status': 'down', 'response_time_ms': 30000, 'details': {'error': 'timeout'}}
        except Exception as e:
            return {'status': 'error', 'response_time_ms': 0, 'details': {'error': str(e)}}

PART 10: CITY-LEVEL DATA FRESHNESS MONITORING
========================================================================================

This is the most important health check. If a city's API stops returning

every day. You need to know within hours, not weeks.

Step A — Create a data freshness check function:

    def check_city_data_freshness(city_slug):
        """
        Query your OWN database (not the external API) to see when the
        most recent permit was ingested for this city.
        """
        # Adjust the query to match your schema — look for the permits/leads
        # table and its city/metro column and created_at/ingested_at timestamp
        result = db.session.execute("""
            SELECT
                COUNT(*) as total_permits,
                MAX(created_at) as newest_permit,
                COUNT(*) FILTER (WHERE created_at >= NOW() - INTERVAL '24 hours') as permits_24h,
                COUNT(*) FILTER (WHERE created_at >= NOW() - INTERVAL '7 days') as permits_7d,
                MIN(created_at) as oldest_permit
            FROM permits
            WHERE city_slug = :city
        """, {'city': city_slug}).fetchone()

        hours_since_newest = 0
        if result.newest_permit:
            hours_since_newest = (datetime.utcnow() - result.newest_permit).total_seconds() / 3600

        # Classify freshness
        if hours_since_newest <= 24:
            status = 'ok'
        elif hours_since_newest <= 72:
            status = 'degraded'     # no new permits in 1-3 days — might be a weekend or API lag
        else:
            status = 'down'         # no new permits in 3+ days — something is broken

        return {
            'status': status,
            'response_time_ms': 0,
            'details': {
                'total_permits': result.total_permits,
                'newest_permit': result.newest_permit.isoformat() if result.newest_permit else None,
                'hours_since_newest': round(hours_since_newest, 1),
                'permits_last_24h': result.permits_24h,
                'permits_last_7d': result.permits_7d
            }
        }

Step B — Create a permit count trend check:

    def check_city_permit_trend(city_slug):
        """
        Compare this week's permit count to last week's. A big drop
        (>50% decline) signals a data pipeline problem.
        """
        result = db.session.execute("""
            SELECT
                COUNT(*) FILTER (WHERE created_at >= NOW() - INTERVAL '7 days') as this_week,
                COUNT(*) FILTER (WHERE created_at >= NOW() - INTERVAL '14 days'
                                 AND created_at < NOW() - INTERVAL '7 days') as last_week
            FROM permits
            WHERE city_slug = :city
        """, {'city': city_slug}).fetchone()

        this_week = result.this_week or 0
        last_week = result.last_week or 1  # avoid division by zero

        pct_change = ((this_week - last_week) / last_week) * 100

        if pct_change >= -20:
            status = 'ok'
        elif pct_change >= -50:
            status = 'degraded'       # noticeable drop, investigate
        else:
            status = 'down'           # >50% drop — pipeline probably broken

        return {
            'status': status,
            'response_time_ms': 0,
            'details': {
                'permits_this_week': this_week,
                'permits_last_week': last_week,
                'pct_change': round(pct_change, 1)
            }
        }

    def run_all_health_checks():
        checks = []

        # 1. City API checks — are the external data sources responding?
        for city_slug, config in CITY_API_CONFIG.items():

            insert_health_check('api_fetch', city_slug, result)

        for city_slug in CITY_API_CONFIG.keys():
            result = check_city_data_freshness(city_slug)
            insert_health_check('data_freshness', city_slug, result)
            checks.append((f'freshness:{city_slug}', result['status']))

        # 3. Permit count trends — any city volume dropping off a cliff?
        for city_slug in CITY_API_CONFIG.keys():
            result = check_city_permit_trend(city_slug)
            insert_health_check('permit_trend', city_slug, result)
            checks.append((f'trend:{city_slug}', result['status']))

        # 4. Route health — is every page on the site responding?
        for route, expected in ROUTES_TO_CHECK:
            result = check_route_health(route, expected)
            insert_health_check('page_load', route, result)
            checks.append((f'route:{route}', result['status']))

        # 5. Third-party services
        for name, fn in [('stripe', check_stripe), ('sendgrid', check_sendgrid), ('database', check_database)]:
            result = fn()
            insert_health_check('service', name, result)
            checks.append((f'service:{name}', result['status']))

        # 6. Alert on failures
        failures = [(name, status) for name, status in checks if status != 'ok']
        if failures:
            send_health_alert(failures)

        return checks

    def insert_health_check(check_type, target, result):
        db.session.execute("""
            INSERT INTO health_checks (check_type, target, status, response_time_ms, details)
            VALUES (:type, :target, :status, :time, :details)
        """, {
            'type': check_type,
            'target': target,
            'status': result['status'],
            'time': result.get('response_time_ms', 0),
            'details': json.dumps(result.get('details', {}))
        })
        db.session.commit()

PART 15: HEALTH DATA RETENTION + CLEANUP
========================================================================================

Health checks run every 30 minutes = ~48 rows per target per day. With
~30 targets (8 cities × 3 checks + 20 routes + 3 services), that's
~1,440 rows/day or ~43,000 rows/month. Not huge, but should be managed.

Add a data retention job:

    def cleanup_old_health_checks():
        """
        Keep detailed data for 30 days, then aggregate to daily summaries.
        Delete raw data older than 90 days.
        """
        # Step 1: Aggregate old data into daily summaries
        db.session.execute("""
            INSERT INTO health_check_daily (check_type, target, check_date,
                         checks_total, checks_ok, checks_degraded, checks_down,
                         avg_response_ms, p95_response_ms, max_response_ms)
            SELECT check_type, target, DATE(created_at),
                   COUNT(*),
                   COUNT(*) FILTER (WHERE status = 'ok'),
                   COUNT(*) FILTER (WHERE status = 'degraded'),
                   COUNT(*) FILTER (WHERE status IN ('down', 'error')),
                   AVG(response_time_ms),
                   PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY response_time_ms),
                   MAX(response_time_ms)
            FROM health_checks
            WHERE created_at < NOW() - INTERVAL '30 days'
            AND DATE(created_at) NOT IN (
                SELECT check_date FROM health_check_daily
                WHERE check_type = health_checks.check_type
                AND target = health_checks.target
            )
            GROUP BY check_type, target, DATE(created_at)
        """)

        # Step 2: Delete raw data older than 30 days (we have daily summaries now)
        db.session.execute("""
            DELETE FROM health_checks
            WHERE created_at < NOW() - INTERVAL '30 days'
        """)

        # Step 3: Delete daily summaries older than 1 year
        db.session.execute("""
            DELETE FROM health_check_daily
            WHERE check_date < CURRENT_DATE - INTERVAL '365 days'
        """)

        db.session.commit()

Create the health_check_daily table:

    CREATE TABLE health_check_daily (
