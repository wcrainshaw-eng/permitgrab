FIX 2: LEAD SCORING — WIDEN DISTRIBUTION TO 40-99  [HIGH]
===============================================================================

PROBLEM:
Lead scores on the main dashboard range from 84-90 (6-point spread). The target
is 40-99 with meaningful differentiation. A $1.8M electrical permit today scores
90, while a $686K HVAC permit from 2 days ago scores 84. That's not enough
spread to help users prioritize.

INTERESTING: The onboarding preview shows scores of 68-70 for smaller permits
($5K-$20K). So the scoring function CAN produce lower scores — but the main
dashboard only shows high-value permits that all cluster at the top.

THE REAL FIX:
The scoring formula needs to use the FULL 40-99 range across the permits that
actually appear on the dashboard. Since the dashboard shows permits ranging
from ~$500K to $1.8M, the formula needs to differentiate within THAT range.

SCORING FORMULA — IMPLEMENT THIS:
```python
def calculate_lead_score(permit):
    score = 0

    # 1. Project value (0-30 points)
    value = permit.get('project_value', 0) or 0
    if value >= 1_000_000:
        score += 30
    elif value >= 500_000:
        score += 25
    elif value >= 250_000:
        score += 20
    elif value >= 100_000:
        score += 15
    elif value >= 50_000:
        score += 10
    elif value >= 10_000:
        score += 5
    else:
        score += 0

    # 2. Recency (0-25 points)
    from datetime import datetime, timedelta
    filed_date = permit.get('filed_date') or permit.get('date')
    if filed_date:
        if isinstance(filed_date, str):
            try:
                filed_date = datetime.strptime(filed_date[:10], '%Y-%m-%d')
            except:
                filed_date = None
        if filed_date:
            days_old = (datetime.now() - filed_date).days
            if days_old <= 1:
                score += 25
            elif days_old <= 3:
                score += 20
            elif days_old <= 7:
                score += 15
            elif days_old <= 14:
                score += 10
            elif days_old <= 30:
                score += 5
            else:
                score += 0

    # 3. Permit status (0-15 points)
    status = (permit.get('status') or '').lower()
    if status in ('active', 'issued', 'approved'):
        score += 15
    elif status in ('permitted', 'filed', 'pending'):
        score += 10
    elif status in ('completed', 'closed'):
        score += 3
    else:
        score += 7

    # 4. Has contact info (0-10 points)
    has_phone = bool(permit.get('phone') or permit.get('contact_phone'))
    has_name = bool(permit.get('contact_name') or permit.get('owner'))
    if has_phone and has_name:
        score += 10
    elif has_phone or has_name:
        score += 5

    # 5. Trade type value (0-10 points) — commercial trades score higher
    trade = (permit.get('trade') or permit.get('permit_type') or '').lower()
    high_value_trades = ['electrical', 'hvac', 'plumbing', 'new construction',
                         'fire protection', 'structural']
    mid_value_trades = ['roofing', 'interior renovation', 'general construction',
                        'demolition', 'addition']
    if any(t in trade for t in high_value_trades):
        score += 10
    elif any(t in trade for t in mid_value_trades):
        score += 7
    else:
        score += 4

    # 6. Jitter for visual variety (0-9 points based on permit ID hash)
    permit_id = str(permit.get('id') or permit.get('permit_number') or '')
    jitter = hash(permit_id) % 10
    score += jitter

    # Clamp to 40-99
    return max(40, min(99, score))
```

KEY POINTS:
- The formula MUST produce scores across the full 40-99 range
- Older, lower-value, completed permits should score in the 40-60 range
- Fresh, high-value, active permits should score in the 80-99 range
- The jitter component prevents identical-looking permits from having the same score

VERIFICATION:
1. Load the main dashboard
2. Check scores across at least 15 permits
3. Confirm the range spans at least 30 points (e.g., lowest ~50, highest ~95)
4. Confirm the onboarding preview also uses the updated scoring

   # One-time cleanup script or migration
