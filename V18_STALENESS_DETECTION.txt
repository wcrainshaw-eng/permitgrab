===============================================================================

it's current. The collector IS running, but the source portals have either
stopped publishing new data or our query filters aren't capturing recent
permits. Meanwhile, users see these cities listed with big permit counts,
click through expecting fresh leads, and get 2-month-old data.

STEP 1: STALENESS DETECTION (automated)
===============================================================================

Add a staleness check that runs after every collection cycle (inside
scheduled_collection() in server.py). This should:

A) Query prod_cities joined with MAX(filing_date) from permits table:

   SELECT pc.city, pc.state, pc.total_permits,
          MAX(p.filing_date) as newest_permit,
          julianday('now') - julianday(MAX(p.filing_date)) as days_stale
   FROM prod_cities pc
   LEFT JOIN permits p ON LOWER(p.city) = LOWER(pc.city)
                       AND LOWER(p.state) = LOWER(pc.state)
   WHERE pc.status = 'active'
   GROUP BY pc.city, pc.state
   HAVING days_stale > 14 OR newest_permit IS NULL
   ORDER BY days_stale DESC

B) For each stale city (>14 days since newest permit):
   - Set a new field on prod_cities: data_freshness = 'stale'
   - Record stale_since date (first time it was detected as stale)
   - Log it: "[Staleness] Atlanta GA: newest permit 58 days old"

C) For each very stale city (>30 days):
   - Set data_freshness = 'very_stale'
   - Auto-pause the city: set status = 'paused' with pause_reason = 'stale_data'
   - This removes it from the website until fresh data is found
   - Log it: "[Staleness] PAUSED Atlanta GA: no new permits in 58 days"

D) Store freshness results in a new table or update prod_cities:

   ALTER TABLE prod_cities ADD COLUMN data_freshness TEXT DEFAULT 'fresh';
   ALTER TABLE prod_cities ADD COLUMN newest_permit_date TEXT;
   ALTER TABLE prod_cities ADD COLUMN stale_since TEXT;
   ALTER TABLE prod_cities ADD COLUMN pause_reason TEXT;


STEP 2: WEBSITE BEHAVIOR FOR STALE/PAUSED CITIES
===============================================================================

A) Paused cities (status='paused', pause_reason='stale_data'):
   - Do NOT show on the cities browse page
   - Do NOT include in sitemap.xml
   - If someone hits the URL directly: show a message like
     "We're updating our data source for Atlanta. Check back soon."
   - Set robots to noindex

B) Stale cities (14-30 days, still active):
   - Show on website but add a subtle indicator:
     "Last updated: Jan 28, 2026" near the permit listings
   - This is honest — users can decide if the data is useful

C) When a paused city gets fresh data again:
   - Auto-reactivate: set status = 'active', data_freshness = 'fresh'
   - Clear pause_reason
   - City reappears on site automatically


STEP 3: AUTOMATIC ALTERNATE SOURCE SEARCH
===============================================================================

When a city is detected as stale (>14 days), automatically trigger the
discovery engine to search for an alternate source. This is a new function:

  find_alternate_source(city, state)

A) Search Socrata catalogs for the city:
   - data.{city_slug}.gov (city's own portal if it exists)
   - data.{state}.gov (state portal, filter by city)
   - Search keywords: "building permits {city}", "construction permits {city}"

B) Search ArcGIS Hub:
   - "building permits {city} {state}"
   - "{county} building permits" (look up the county for the city)

C) Search the county portal:
   - Many cities don't publish their own data, but the county does
   - For Atlanta: look at Fulton County
   - For Dallas: look at Dallas County
   - For Denver: look at Denver County (same entity but may have different portal)

D) Check if there's a different endpoint on the SAME source:
   - Sometimes open data portals change their dataset IDs
   - The old dataset may be archived while a new one exists
   - Search the portal for alternative datasets

E) If a new source is found:
   - Add to discovered_sources table
   - Test it via test_socrata_endpoint() or test_arcgis_endpoint()
   - If it returns permits with recent dates:
     * Update the city's source info in prod_cities
     * Run a collection for that city
     * Log: "[Recovery] Found alternate source for Atlanta GA: {url}"
   - If no alternate found:
     * Log: "[Recovery] No alternate source found for Atlanta GA"
     * Add to a manual_review queue (see Step 5)

STEP 5: MANUAL REVIEW QUEUE (stale_cities_review table)
===============================================================================

For cities where no automated alternate source is found, track them in a
new table so you know what still needs human research:

  CREATE TABLE stale_cities_review (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    city TEXT NOT NULL,
    state TEXT NOT NULL,
    original_source TEXT,
    last_permit_date TEXT,
    stale_since TEXT,
    auto_search_attempted BOOLEAN DEFAULT FALSE,
    auto_search_result TEXT,     -- 'found_alternate' / 'no_alternate'
    manual_notes TEXT,           -- for your notes when you research
    alternate_source_url TEXT,   -- if you find one manually
    status TEXT DEFAULT 'needs_review',  -- needs_review / in_progress / resolved / no_source
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
  );

Add an admin endpoint: /api/admin/stale-cities that returns this queue.

  FRESHNESS_FRESH     = 0-14 days    -> status stays 'active', show normally

These thresholds should be configurable via system_state table so you can
adjust without code changes.
