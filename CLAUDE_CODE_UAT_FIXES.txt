PART 8: FIX TRADE CLASSIFICATION BADGES
------------------------------------------------------------------------

Problem: Every single permit card shows "General Construction" as the trade
badge, even when the permit metadata clearly contains the correct trade
(Electrical Permit, Demolition, Plumbing/Gas, Grading, etc.).

The data is there — it's just not being mapped to the badge display.

Find where permit cards render the trade badge (look for "General Construction"
as a default/fallback string). Then:

1. Check where permit data comes from (Socrata API / ArcGIS FeatureServer).
   The permit type or description field should contain the actual trade.

2. Build a mapping function that classifies permits into trades based on
   their type/description fields:
   - If permit_type contains "electrical" → "Electrical"
   - If permit_type contains "plumbing" or "gas" → "Plumbing/Gas"
   - If permit_type contains "mechanical" or "hvac" → "HVAC/Mechanical"
   - If permit_type contains "demolition" → "Demolition"
   - If permit_type contains "roofing" → "Roofing"
   - If permit_type contains "grading" or "earthwork" → "Grading"
   - If permit_type contains "fire" or "sprinkler" → "Fire Protection"
   - If permit_type contains "solar" or "photovoltaic" → "Solar"
   - If permit_type contains "sign" → "Signage"
   - If description contains "tenant improvement" or "TI" → "Tenant Improvement"
   - If description contains "renovation" or "remodel" → "Renovation"
   - If description contains "new construction" or "new build" → "New Construction"
   - If description contains "addition" → "Addition"
   - Fallback: "General Construction" (only if nothing else matches)

3. Use this mapping when rendering the trade badge on every permit card.

4. Also use this same mapping for the filter dropdowns so users can filter
   by these more specific trades.

Test: Browse permits and verify you see a variety of trade badges (not just
"General Construction" on everything). Spot-check a few: an electrical permit
should show "Electrical", a demolition permit should show "Demolition", etc.

PART 9: WIDEN LEAD SCORING RANGE
------------------------------------------------------------------------

Problem: Every permit scores between 83-88 points. There's no meaningful
differentiation between a $2.3M new construction filed yesterday and a
$958K renovation from 2 weeks ago. Contractors can't use the score to
prioritize their outreach.

Find the lead scoring function/algorithm. It should be weighting factors
like: project value, permit recency, contact info completeness, permit
status, project type, etc.

Redesign the scoring to produce a wider distribution (40-99 range):

VALUE COMPONENT (0-30 pts):
  - $0-100K     →  5 pts
  - $100K-500K  → 10 pts
  - $500K-1M    → 15 pts
  - $1M-5M      → 20 pts
  - $5M+        → 30 pts
  - Unknown     →  8 pts (don't penalize too much)

RECENCY COMPONENT (0-25 pts):
  - Filed today      → 25 pts
  - Filed 1-3 days   → 20 pts
  - Filed 4-7 days   → 15 pts
  - Filed 1-2 weeks  → 10 pts
  - Filed 2-4 weeks  →  5 pts
  - Filed 1+ month   →  2 pts

CONTACT INFO COMPONENT (0-20 pts):
  - Has phone number      → +10 pts
  - Has email              → +5 pts
  - Has owner/company name → +5 pts
  - No contact info at all →  0 pts

STATUS COMPONENT (0-15 pts):
  - "Issued" / "Approved"  → 15 pts (ready to start, highest intent)
  - "Under Review"         → 10 pts
  - "Submitted" / "Filed"  →  8 pts
  - "In Review"            →  5 pts
  - "Expired" / "Voided"   →  0 pts

PROJECT TYPE BONUS (0-9 pts):
  - New Construction        → +9 pts
  - Major Renovation (>$500K) → +7 pts
  - Tenant Improvement      → +5 pts
  - Addition                → +4 pts
  - Minor Renovation        → +2 pts
  - Other                   → +1 pt

Max possible = 99 pts. Min realistic = ~40 pts.

This means:
  - A $3M new construction filed today with full contact and "Issued" status → 95+
  - A $200K renovation filed 3 weeks ago with no phone → ~55
  - Contractors can immediately see which leads to call first

Test: Browse the dashboard and verify scores now range from ~40 to ~95+ with
meaningful variation. Sort by score — the top leads should clearly be higher
value and more recent than the bottom ones.
