ISSUE #1: TRADE PAGES SHOW 0 PERMITS (Blocking 1,836 pages)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Severity: CRITICAL
Effort: 2-4 hours
Impact: Unblocks 1,836 pages from "doorway page" penalty

Problem:
  - Code built /permits/[city]/[trade] pages for all combinations
  - Pages render but show "0 permits", "$N/A value"
  - Example: /permits/houston/plumbing has 0 permits
  - Houston DB has 9,000+ permits; plumbing subset should be hundreds

Root Cause:
  - DB trade_category values don't match URL slugs exactly
  - No fuzzy/LIKE-based matching implemented
  - Example: DB has "Plumbing - Residential", "Plumbing (Interior)"
    But query looks for exact match "plumbing"

Solution:
  1. Audit DB: SELECT DISTINCT trade_category FROM permits LIMIT 50
  2. Create mapping: "plumbing" → ["Plumbing%", "Plumbing - Residential", ...]
  3. Implement LIKE-based query in trade page route
  4. Test: Verify houston/plumbing shows real permits

SEO Impact:
  - Without fix: 1,836 thin-content pages = domain penalty
  - With fix: 1,836 data-rich pages contributing to rankings
