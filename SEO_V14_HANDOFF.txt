ITEM 1 — CRITICAL: FIX TRADE PAGE DATA (0 PERMITS BUG)
=====================================================
Priority: HIGHEST — Do this first. Everything else depends on it.

PROBLEM:
  All 1,836 trade-specific landing pages show zero data.
  Example: /permits/houston/plumbing displays:
    - "0 This Month"
    - "0 This Week"
    - "$N/A Avg Value"
    - No recent permits listed in the table

  Houston has 9,000+ total permits. Plumbing should have hundreds.

DIAGNOSIS STEPS:
  1. Run this query to see what trade values exist in the DB:
       SELECT DISTINCT description, COUNT(*) as cnt
       FROM permits
       WHERE LOWER(city) = 'houston'
       GROUP BY description
       ORDER BY cnt DESC
       LIMIT 30;

     Also try:
       SELECT DISTINCT work_type, COUNT(*) as cnt
       FROM permits
       WHERE LOWER(city) = 'houston'
       GROUP BY work_type
       ORDER BY cnt DESC
       LIMIT 30;

     (The column might be called "description", "work_type", "trade",
     "permit_type", or "category" — check the schema.)

  2. Compare what the DB returns to what the URL slug expects.
     The route is: /permits/<city_slug>/<trade_slug>
     Trade slugs in the sitemap are: plumbing, electrical, hvac,
     roofing, general-construction, solar, demolition, painting

  3. The mismatch is likely one of:
     a) The DB column stores full descriptions like "Plumbing -
        New Installation" and the query filters on exact match
        "plumbing" — fix with ILIKE '%plumbing%' or a mapping dict
     b) The route handler doesn't query the DB at all — it renders
        the template with empty data
     c) The trade filter function exists but isn't called in the route

FIX:
  In server.py, find the route for /permits/<city_slug>/<trade_slug>.
  Ensure it:
    1. Maps the URL trade_slug to a DB query filter
    2. Queries permits filtered by city AND trade
    3. Passes the filtered permit list to the template
    4. Calculates this_month_count, this_week_count, avg_value
       from the filtered results

  Recommended: Create a TRADE_MAPPING dict:
    TRADE_MAPPING = {
        'plumbing': ['plumbing', 'plumb'],
        'electrical': ['electrical', 'electric', 'elec'],
        'hvac': ['hvac', 'heating', 'air conditioning', 'mechanical'],
        'roofing': ['roofing', 'roof', 'reroof', 're-roof'],
        'general-construction': ['general', 'addition', 'alteration',
            'remodel', 'renovation', 'new construction'],
        'solar': ['solar', 'photovoltaic', 'pv'],
        'demolition': ['demolition', 'demo'],
        'painting': ['painting', 'paint']
    }

  Then filter with:
    keywords = TRADE_MAPPING.get(trade_slug, [trade_slug])
    # Build WHERE clause: description ILIKE ANY of the keywords

  SAFETY: If the data fix is complex and you can't get real data
  flowing quickly, add this to the trade page template as a stopgap:
    <meta name="robots" content="noindex, follow">
  This prevents Google from indexing 1,836 empty pages. Remove
  the noindex once data populates.

VERIFY: After fix, load /permits/houston/plumbing and confirm it
shows actual permit counts and recent permits in the table.
