PART 2: DYNAMIC CITY × TRADE ROUTES
=============================================

Create TWO dynamic route patterns:

A) /permits/<city-slug>/<trade-slug>
   Example: /permits/nashville/plumbing

B) /permits/<city-slug> (already exists from previous SEO batch — make sure it still works)

The city × trade route should:

1. Look up the city from whatever city registry/config the app uses to get the list of active cities. If the city slug doesn't match an active city, return 404.

2. Look up the trade from TRADE_REGISTRY. If the trade slug doesn't match, return 404.

3. Query the local permit data for that city, filtering permits whose description/type matches any of the trade's keywords (case-insensitive substring match).

4. Render a template (templates/city_trade_landing.html) with:

   META TAGS:
   - Title: "{Trade} Permits in {City}, {State} | PermitGrab"
     Example: "Plumbing Permits in Nashville, TN | PermitGrab"
   - Description: "Track new {trade description_template} in {City}, {State}. Real-time {trade name} contractor leads from {City} building permits. Updated daily."
   - Canonical URL: https://permitgrab.com/permits/{city-slug}/{trade-slug}
   - Open Graph tags (og:title, og:description, og:url, og:type=website)

   PAGE CONTENT:
   - H1: "{Trade} Permits in {City}, {State}"
   - Intro paragraph (auto-generated): "PermitGrab tracks all new {trade description_template} in {City}, {State}. {Buyer_phrase} use this data to find new jobs before the competition. Below are the latest {trade name} permits filed in {City}."

   - STATS BOX: Show counts pulled from actual data:
     * Total {trade} permits this month
     * Total {trade} permits this week
     * Average estimated value (if value data available)
     * "Updated daily" badge

   - RECENT PERMITS TABLE: Show the 10 most recent permits matching this trade in this city. Columns: Date, Address, Description, Estimated Value, Status. If user is not logged in, blur the last 3 rows and show "Sign up free to see all {trade} permits in {City}" CTA.

   - FAQ SECTION (for FAQPage schema): Auto-generate 4 FAQs:
     * "How many {trade} permits are filed in {City} per month?" → Answer with real count from data
     * "What types of {trade} work require permits in {City}?" → Answer listing the trade's keyword categories
     * "How can I find {trade} leads in {City}?" → Answer mentioning PermitGrab's daily updates and lead scoring
     * "How quickly are {trade} permits updated on PermitGrab?" → Answer: "Permits are collected daily from {City}'s open data portal and available on PermitGrab within 24 hours of filing."

   - CROSS-LINKS SECTION:
     * "Other trades in {City}:" → links to all other trade pages for this city
     * "See {Trade} permits in other cities:" → links to this trade in all other active cities
     * Link back to /permits/{city-slug} (the main city page)

   - CTA BOX: "Get all {City} {Trade} permits delivered to your dashboard. Start free →"

   STRUCTURED DATA (JSON-LD):
   - FAQPage schema with the 4 FAQs above
   - BreadcrumbList: Home > {City} Permits > {Trade} Permits

5. IMPORTANT: The route must dynamically discover active cities from the app's existing city configuration — do NOT hardcode city names. When Code adds a new city, that city's trade pages should automatically work without any changes to this code.

PART 5: INTERNAL LINKING MESH
=============================================

On EVERY page in the app, add contextual cross-links:

1. City pages (/permits/{city}) should link to all trade sub-pages for that city in a "Browse by Trade" section at the bottom.

2. Trade pages (/permits/{city}/{trade}) should link to:
   - The parent city page
   - Other trades in the same city
   - The same trade in neighboring/related cities

3. The dashboard should have a "Popular Searches" or "Trending Permits" widget in the sidebar linking to the highest-volume city×trade combinations (auto-calculated from actual data).

This internal linking mesh is critical for SEO — it helps Google discover and index all pages, and distributes page authority across the site.
