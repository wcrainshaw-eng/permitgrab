ROOT CAUSE: get_cities_with_data() DEDUP IS INCOMPLETE
================================================================================

The function `get_cities_with_data()` at server.py line 1578 is THE single data source
that feeds:
  - Nav dropdown: templates/partials/nav.html line 176 (nav_cities[:10])
  - Dashboard dropdown: templates/dashboard.html line 1363 (footer_cities[:20])
  - /cities top cards: server.py line 5852 (top_cities = all_cities[:12])
  - /cities "All Cities by State": server.py line 5847 (sorted_states)
  - Footer links on every page

The V13.1 dedup (lines 1602-1670) groups by (name_lower, state_lower) and sums permit
counts. But it has these gaps:

  1. Same city with DIFFERENT state values are NOT merged — e.g., "Houston" with state
     "TX" and "Houston" with state "OK" remain as separate entries. The DB has some
     permits tagged with wrong states from bulk sources.

  2. State names are not filtered out — "California", "New Jersey", "Nevada" exist as
     city entries in the permits DB (from bulk sources using state-level data or bad
     city_field mappings).

  3. Non-US entries are not filtered — Canadian cities (Calgary AB, Edmonton AB) exist
     in the permits DB.

  4. The dedup only merges entries with identical (city_lower, state_lower). But bulk
     sources sometimes store the SAME city under slightly different entries that don't
     get merged because the state field differs — resulting in "Austin" appearing 6
     times under TX with different permit counts (8730, 4154, 4035, 57, 1, 1).

FIX get_cities_with_data() ONCE and it cascades everywhere. Here's what to add:

  a) Filter: skip entries where city name is a US state name
  b) Filter: skip entries where state code is not a valid US state/territory
  c) Dedup: after the current (name_lower, state_lower) grouping, do a SECOND pass
     that merges entries with the same name_lower across ALL states — pick the state
     with the highest permit count. This handles "Houston OK" + "Houston TX" and the
     6x Austin problem.
  d) Add a VALID_US_STATES set and a US_STATE_NAMES set for these filters.
