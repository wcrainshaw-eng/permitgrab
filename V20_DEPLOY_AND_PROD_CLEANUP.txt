STEP 5: DEDUPLICATE PERMITS
------------------------------------------------------------------------

This is the biggest cleanup. Remove duplicate rows where the same
address + city + state + filing_date appears multiple times. Keep the
row with the highest estimated_cost (most useful lead).

WARNING: This query can be slow on 500k+ rows. Run during off-peak hours.

python3 << 'PYEOF'
import db
db.init_db()
conn = db.get_connection()

# Count before
before = conn.execute("SELECT COUNT(*) FROM permits").fetchone()[0]
print(f"Permits before dedup: {before}")

# Find duplicates - keep the one with highest estimated_cost per group
# Strategy: delete all but the "best" row in each duplicate group
# Using rowid since permits table may not have an auto-increment id

# First, count how many excess we'll remove
excess = conn.execute("""
    SELECT SUM(cnt - 1) FROM (
        SELECT COUNT(*) as cnt FROM permits
        WHERE address IS NOT NULL AND address != '' AND address != 'Address not provided'
        GROUP BY address, city, state, filing_date
        HAVING COUNT(*) > 1
    )
""").fetchone()[0] or 0
print(f"Excess duplicate rows to remove: {excess}")

if excess > 0:
    # Delete duplicates, keeping the row with highest estimated_cost
    # (or first row if costs are equal)
    deleted = conn.execute("""
        DELETE FROM permits
        WHERE rowid NOT IN (
            SELECT MIN(rowid)
            FROM permits
            GROUP BY address, city, state, filing_date
        )
        AND address IS NOT NULL
        AND address != ''
        AND address != 'Address not provided'
    """).rowcount
    conn.commit()
    print(f"Deleted {deleted} duplicate rows")

after = conn.execute("SELECT COUNT(*) FROM permits").fetchone()[0]
print(f"Permits after dedup: {after}")
print(f"Removed: {before - after}")
PYEOF

NOTE: The above dedup keeps one row per (address, city, state, filing_date)
group. This means a building with both an electrical permit and a plumbing
permit filed on the same day at the same address will be deduped to ONE row.
This is intentional — for lead generation, the contractor cares about the
ADDRESS not the individual permit sub-types. If you want to preserve
different permit_types, change GROUP BY to include permit_type:
  GROUP BY address, city, state, filing_date, permit_type

import db
db.init_db()
conn = db.get_connection()

# Recalculate total_permits for each city
cities = conn.execute("""
    SELECT city, state, city_slug FROM prod_cities WHERE status='active'
""").fetchall()

updated = 0
for city_row in cities:
    city = city_row['city']
    state = city_row['state']
    slug = city_row['city_slug']

    # Count permits
    total = conn.execute(
        "SELECT COUNT(*) FROM permits WHERE city=? AND state=?",
        (city, state)
    ).fetchone()[0]

    # Count last 30 days
    last_30 = conn.execute("""
        SELECT COUNT(*) FROM permits
        WHERE city=? AND state=?
        AND filing_date >= date('now', '-30 days')
    """, (city, state)).fetchone()[0]

    # Update prod_cities
    conn.execute("""
        UPDATE prod_cities
        SET total_permits=?, permits_last_30d=?
        WHERE city_slug=?
    """, (total, last_30, slug))
    updated += 1

conn.commit()
print(f"Recalculated stats for {updated} cities")

# Summary
print("\nTop 20 cities by permit count:")
top = conn.execute("""
    SELECT city, state, total_permits, permits_last_30d
    FROM prod_cities
    WHERE status='active'
    ORDER BY total_permits DESC
    LIMIT 20
""").fetchall()
for t in top:
    print(f"  {t['total_permits']:>6}  {t['city']}, {t['state']}  (last 30d: {t['permits_last_30d']})")

NEW DUPLICATES FROM NEW CITIES — ROOT CAUSE
================================================================================

The new cities are generating duplicates because their source data has
multiple permits for the same address on the same day (different permit
types: electrical, plumbing, mechanical, structural, etc. for one
building project). This is normal in the source data but creates noise
in a lead generation product.

To prevent future duplicates, the collector should either:

A) SKIP ON INSERT: Before inserting, check if (address, city, state,
   filing_date) already exists. If so, update the existing row's
   estimated_cost to the MAX of old and new values.

B) UNIQUE CONSTRAINT: Add a UNIQUE constraint or index on
   (address, city, state, filing_date) and use INSERT OR REPLACE.

C) DISPLAY-LEVEL DEDUP: Keep all rows but GROUP BY in the API query
   that feeds the homepage. This preserves granular data while showing
   clean results.

Option A is simplest and most robust. Add to collector.py:

   # Before INSERT
   existing = conn.execute(
       "SELECT rowid FROM permits WHERE address=? AND city=? AND state=? AND filing_date=?",
       (address, city, state, filing_date)
   ).fetchone()
   if existing:
       # Update if new cost is higher
       conn.execute(
           "UPDATE permits SET estimated_cost = MAX(estimated_cost, ?) WHERE rowid=?",
           (estimated_cost, existing[0])
       )
