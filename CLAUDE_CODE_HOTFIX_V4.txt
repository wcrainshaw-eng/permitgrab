FIX 2: LEAD SCORING — NORMALIZE TO VISIBLE DATASET  [MEDIUM]
===============================================================================

PROBLEM:
The V3 scoring formula (commit b8818bc) produces scores in the 79-99 range for
permits visible on the dashboard. The target was 40-99. The formula itself is
mathematically correct, but the data characteristics of dashboard permits
(all recent, mostly active/issued, with contact info, $1K-$1.8M value) mean
the minimum score is approximately:
  - Value: 5-30 pts (most are $1K+ so 5-30)
  - Recency: 15-25 pts (all within 7 days)
  - Status: 10-15 pts (active/issued/permitted)
  - Contact: 5-10 pts (most have contact info)
  - Trade: 4-10 pts
  - Jitter: 0-9 pts
  = Minimum ~54, but in practice ~79 because most permits tick all boxes

THE REAL ISSUE:
The formula uses ABSOLUTE thresholds that don't differentiate within the
dashboard's data range. A $7,500 plumbing permit from 5 days ago with contact
info scores 79, and a $1.8M electrical permit from today with contact info
scores 99. That's only 20 points of spread.

BETTER APPROACH — PERCENTILE-BASED SCORING:
Instead of absolute thresholds, normalize scores relative to the permits
currently in the dataset. This guarantees spread across the full range.

```python
def calculate_lead_scores(permits):
    """Score permits relative to each other for maximum spread."""
    import hashlib

    # 1. Calculate raw composite score for each permit
    raw_scores = []
    for p in permits:
        raw = 0

        # Value component (log scale for better distribution)
        value = p.get('project_value') or p.get('estimated_cost') or 0
        if value > 0:
            import math
            raw += math.log10(max(value, 1)) * 5  # log scale: $1K=15, $100K=25, $1M=30

        # Recency (days old, inverted)
        from datetime import datetime
        filed = p.get('filed_date') or p.get('filing_date') or p.get('date')
        if filed:
            if isinstance(filed, str):
                try: filed = datetime.strptime(filed[:10], '%Y-%m-%d')
                except: filed = None
            if filed:
                days = (datetime.now() - filed).days
                raw += max(0, 30 - days)  # 30 pts for today, 0 for 30+ days

        # Status
        status = (p.get('status') or '').lower()
        status_scores = {'filed': 15, 'active': 12, 'issued': 10, 'approved': 10,
                         'permitted': 8, 'pending': 5, 'completed': 2, 'closed': 1}
        raw += status_scores.get(status, 5)

        # Contact info
        has_phone = bool(p.get('phone') or p.get('contact_phone'))
        has_name = bool(p.get('contact_name') or p.get('owner') or p.get('owner_name'))
        raw += (5 if has_phone else 0) + (5 if has_name else 0)

        # Trade type
        trade = (p.get('trade') or p.get('permit_type') or p.get('trade_category') or '').lower()
        high_trades = ['electrical', 'hvac', 'plumbing', 'new construction', 'fire protection']
        raw += 10 if any(t in trade for t in high_trades) else 5

        raw_scores.append(raw)

    # 2. Normalize to 40-99 range using percentile ranking
    if not raw_scores:
        return []

    sorted_scores = sorted(set(raw_scores))
    n = len(sorted_scores)

    final_scores = []
    for i, raw in enumerate(raw_scores):
        if n <= 1:
            percentile = 0.5
        else:
            rank = sorted_scores.index(raw)
            percentile = rank / (n - 1)

        # Map percentile to 40-99
        score = int(40 + percentile * 59)

        # Add small jitter from permit ID
        permit_id = str(permits[i].get('id') or permits[i].get('permit_number') or '')
        jitter = int(hashlib.md5(permit_id.encode()).hexdigest()[:4], 16) % 5
        score = max(40, min(99, score + jitter - 2))

        final_scores.append(score)

    return final_scores
```

KEY CHANGES:
- Use PERCENTILE-BASED scoring instead of absolute thresholds
- This guarantees the full 40-99 range is used regardless of data characteristics
- The best permit in any dataset gets ~99, the worst gets ~40
- Use log scale for project value to better differentiate within ranges
