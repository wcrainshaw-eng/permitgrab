# V12.57 Deploy + Full System Test Plan

## What's broken right now (and what this deploy fixes)

| Issue | Root Cause | Fix |
|-------|-----------|-----|
| Discovery engine onboards 0 cities | County-matching penalizes datasets from county's own domain (e.g., `datacatalog.cookcountyil.gov` rejected for Cook County) | Smart domain matching: skip penalty if domain contains county name, light penalty for state portals |
| REFRESH collector never runs (0 deltas in 24+ hrs) | Every deploy starts a 365-day full collection that holds a file lock forever. Subsequent REFRESH attempts SKIP. Instances get killed on deploy before lock expires. | Replaced 365-day full rebuild with 7-day refresh on startup. Clear orphaned lock file at boot. Reduced stale timeout from 2hr to 30min. |
| Daily digest email not sent | `users.watched_competitors` column doesn't exist in Postgres — crashes SQLAlchemy query | Auto-migrate missing columns at startup with `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` |
| Addresses show raw Socrata JSON | `parse_address_value()` didn't handle GeoJSON Point objects (`{'type': 'Point', 'coordinates': [...]}`) | Added GeoJSON handling. Also fixed description generator to clean JSON from addresses. |
| Descriptions show raw JSON | `generate_permit_description()` used raw address without cleaning | Added `parse_address_value()` call in description generator |
| Canadian permits still in DB (119,020 not 114,972) | Cleanup ran on Render shell but lock/deploy churn may have prevented commit | Re-run cleanup after deploy (see Render commands below) |
| `_fix_socrata_addresses()` crashes on startup | Called `permitdb.get_db()` which doesn't exist | Changed to `sqlite3.connect('/var/data/permitgrab.db')` |

# === 4. FIX BAD ADDRESSES + DESCRIPTIONS IN DB ===
python3 -c "
import sqlite3, ast, json, re

conn = sqlite3.connect('/var/data/permitgrab.db')
conn.row_factory = sqlite3.Row

# Find all permits with JSON in address
addr_rows = conn.execute(\"SELECT permit_number, address, zip FROM permits WHERE address LIKE '%{%'\").fetchall()
print(f'Found {len(addr_rows)} permits with JSON in address')

fixed = 0
for row in addr_rows:
    pn, raw, z = row['permit_number'], row['address'], row['zip'] or ''
    try:
        parsed = ast.literal_eval(raw)
        clean = ''
        new_zip = z
        if isinstance(parsed, dict):
            if parsed.get('type') == 'Point':
                clean = ''  # GeoJSON, no address
            else:
                human = parsed.get('human_address', '')
                if human:
                    if isinstance(human, str):
                        human = json.loads(human)
                    if isinstance(human, dict):
                        clean = human.get('address', '').strip()
                        if not new_zip:
                            new_zip = human.get('zip', '')
        if clean != raw:
            conn.execute('UPDATE permits SET address=?, zip=? WHERE permit_number=?', (clean, new_zip or '', pn))
            fixed += 1
    except:
        pass
conn.commit()
print(f'Fixed {fixed} addresses')

# Fix descriptions with JSON
desc_rows = conn.execute(\"SELECT permit_number, display_description FROM permits WHERE display_description LIKE '%human_address%' OR display_description LIKE '%latitude%' OR display_description LIKE '%coordinates%'\").fetchall()
print(f'Found {len(desc_rows)} permits with JSON in description')
dfixed = 0
for row in desc_rows:
    pn, desc = row['permit_number'], row['display_description'] or ''
    cleaned = re.sub(r\"\{[^}]*'(?:human_address|latitude|type)'[^}]*\}\", '', desc)
    cleaned = re.sub(r'\s+', ' ', cleaned).strip()
    if cleaned != desc:
        conn.execute('UPDATE permits SET display_description=? WHERE permit_number=?', (cleaned, pn))
        dfixed += 1
conn.commit()
print(f'Fixed {dfixed} descriptions')
conn.close()
"
