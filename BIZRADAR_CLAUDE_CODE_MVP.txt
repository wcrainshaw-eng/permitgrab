PART 2: STATE DATA REGISTRY (state_configs.py)
=============================================

Create a STATE_REGISTRY dictionary. Each state entry contains:
{
    "name": "Florida",
    "abbreviation": "FL",
    "slug": "florida",
    "collector_type": "ftp",           # ftp | api | bulk | opencorporates
    "data_url": "ftp://...",           # Primary data source URL
    "update_frequency": "daily",       # daily | weekly | monthly
    "data_format": "csv",             # csv | json | txt | xml
    "field_map": {
        "entity_name": "...",
        "entity_type": "...",
        "formation_date": "...",
        "registered_agent": "...",
        "principal_address": "...",
        "filing_number": "...",
        "status": "...",
        "purpose": "..."
    },
    "cost": "free",                    # free | $X per download
    "active": true,
    "tier": 1                          # 1=launch, 2=phase2, 3=phase3
}

MVP launch states (Tier 1):

1. FLORIDA
   - Type: ftp
   - URL: https://dos.fl.gov/sunbiz/other-services/data-downloads/daily-data/
   - FTP credentials: Public access (check the data downloads page for current FTP URL and login)
   - Format: Fixed-width text files OR CSV (check format on download page)
   - Contains: Corp/LLC/LP filings added that day
   - Fields: Entity name, entity number, filing date, status, state, registered agent name/address, principal address
   - Cost: Free
   - Update: Daily (workdays)

2. NEW YORK
   - Type: api / bulk
   - URL: https://data.ny.gov/ — Dataset "Corporations and Other Entities: All Filings"
   - Also has 30-day rolling window dataset
   - Format: JSON (Socrata SODA API) or CSV download
   - Fields: DOS ID, Filing Date, Entity Name, County, Jurisdiction, Entity Type
   - Cost: Free
   - Update: Regularly updated

3. CALIFORNIA
   - Type: api
   - URL: bizfileonline.sos.ca.gov — API Management Portal
   - Also: Bulk "Master Unload" for $100 one-time
   - Format: JSON (REST API)
   - Fields: Entity name, number, registration date, address
   - Cost: API free with subscription key; Bulk $100
   - Notes: Register for API key at their developer portal

4. TEXAS
   - Type: bulk
   - URL: https://direct.sos.state.tx.us/ (SOSDirect)
   - Bulk data order from the website
   - Format: Electronic data files
   - Cost: Varies (bulk orders have fees)
   - Notes: May need to request bulk access

5. WASHINGTON
   - Type: bulk
   - URL: sos.wa.gov — business entity data
   - Format: CSV/bulk download
   - Cost: Free

6. KENTUCKY
   - Type: bulk
   - URL: sos.ky.gov/bus/Pages/Bulk-Data-Service.aspx
   - Format: Bulk subscription
   - Cost: Monthly subscription (commercial use)

For states without direct access, use:
7. OPENCORPORATES FALLBACK
   - Type: api
   - URL: https://api.opencorporates.com/v0.4/
   - Endpoint: /companies/search?q=*&jurisdiction_code=us_{state}&created_since={yesterday}
   - Format: JSON
   - Cost: Free for limited queries, paid for commercial volume
   - Notes: Good fallback for any state. Rate limited.

PART 3: LEAD DATA MODEL (models/leads.py)
=============================================

Each filing becomes a lead record:
{
    "lead_id": "lead_fl_L24000012345",
    "entity_name": "Sunrise Coffee LLC",
    "entity_type": "LLC",
    "state": "FL",
    "state_name": "Florida",
    "formation_date": "2026-03-15",
    "date_collected": "2026-03-16",
    "days_since_filing": 1,
    "registered_agent": {
        "name": "John Smith",
        "address": "123 Main St, Miami, FL 33101"
    },
    "principal_address": "456 Ocean Dr, Miami Beach, FL 33139",
    "filing_number": "L24000012345",
    "status": "Active",
    "purpose_statement": "Coffee shop and bakery",
    "industry_tag": "restaurant",
    "industry_confidence": 0.85,
    "contact": {
        "phone": null,
        "email": null,
        "linkedin": null
    },
    "lead_score": 72,
    "freshness_badge": "NEW"  // NEW (<48hrs), RECENT (<7 days), STANDARD (>7 days)
}

Functions to implement:
- store_leads(leads_list) — deduplicate by filing_number + state, store to data/leads.json
- get_leads(filters) — query with filters: state, entity_type, date_range, keyword, industry_tag, min_score
- get_lead(lead_id) — single lead detail
- get_lead_stats() — counts by state, entity type, industry, and freshness
- cleanup_old_leads() — archive leads older than 90 days to reduce active dataset size

PART 4: COLLECTORS (collectors/)
=============================================

base_collector.py:
- Abstract base class with methods: fetch(), parse(), normalize(), store()
- Logging to data/collection_stats.json
- Error handling with retry logic (3 retries, exponential backoff)
- Rate limiting (1 second between requests)

ftp_collector.py (for Florida):
- Download daily data file from SunBiz FTP
- Parse the file format (check SunBiz documentation for current format)
- Extract new filings only (filter by filing date = today or yesterday)
- Normalize to standard lead format
- Store via models/leads.py

api_collector.py (for NY, CA, OpenCorporates):
- REST API calls with appropriate authentication
- Pagination handling
- Filter by formation_date > last_collection_date
- Normalize to standard lead format

bulk_collector.py (for TX, WA, KY):
- Download bulk files
- Diff against previous download to find new filings
- Normalize to standard lead format

enrichment.py:
- classify_industry(entity_name, purpose_statement) — Simple keyword-based classification:
  * If name contains "restaurant|cafe|coffee|bar|grill|pizza|bakery|catering" → "restaurant"
  * If name contains "construction|building|roofing|plumbing|electric" → "construction"
  * If name contains "consulting|advisory|solutions|services" → "professional_services"
  * If name contains "medical|dental|health|therapy|clinic" → "healthcare"
  * If name contains "real estate|realty|property|homes" → "real_estate"
  * If name contains "tech|software|digital|app|data|cyber" → "technology"
  * If name contains "salon|spa|beauty|barber|aesthetic" → "beauty"
  * If name contains "fitness|gym|training|yoga|crossfit" → "fitness"
  * If name contains "retail|shop|store|boutique|market" → "retail"
  * If name contains "transport|trucking|logistics|moving|freight" → "transportation"
  * If name contains "cleaning|janitorial|maid|pressure wash" → "cleaning"
  * If name contains "landscape|lawn|garden|tree" → "landscaping"
  * Default → "other"
  (Phase 2 will upgrade this to an AI classifier)

- score_lead(lead) — Score 0-100:
  * +25 if entity_type is LLC (most common small business)
  * +20 if formation_date is within 48 hours
  * +15 if formation_date is within 7 days
  * +10 if industry_tag is not "other" (classified = more actionable)
  * +15 if principal_address is present and complete
  * +10 if registered_agent name looks like a person (not a registered agent service)
  * +5 if purpose_statement is present
  * +10 if state is high-activity (FL, TX, CA, NY)
