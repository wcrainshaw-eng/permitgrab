STEP 3: HARVEST CITIES FROM PERMITS TABLE → prod_cities (20 min)
===================================================================

This is the money step. Query the permits table to find every unique
(city, state) pair that has actual permit data, then bulk-insert
into prod_cities.

SQL approach:

  -- Find all cities with recent permits (last 90 days)
  SELECT
    p.city,
    p.state,
    COUNT(*) as total_permits,
    COUNT(CASE WHEN p.date >= date('now', '-30 days') THEN 1 END) as permits_30d,
    MAX(p.date) as last_permit_date,
    MIN(p.date) as first_permit_date
  FROM permits p
  WHERE p.city IS NOT NULL
    AND p.city != ''
    AND p.state IS NOT NULL
    AND LENGTH(p.city) > 1
  GROUP BY p.city, p.state
  HAVING total_permits >= 1
  ORDER BY total_permits DESC;

For each result:
  A) Skip if already in prod_cities
  B) Generate city_slug as: lower(city)-lower(state) with spaces→hyphens
  C) Try to match against us_cities table to get county, population
  D) Determine source_type and source_scope:
     - If city came from a bulk source → source_scope = 'county' or 'state'
     - If city came from a city-level source → source_scope = 'city'
  E) INSERT into prod_cities with:
     - status = 'active'
     - added_by = 'bulk_harvest'
     - verified_date = datetime('now')
     - total_permits = count from query
     - permits_last_30d = count from query
     - notes = 'Harvested from existing bulk source collection'

Also update us_cities table:
  - Set status = 'covered_by_source' for matched cities
  - Set covered_by_source = the bulk source key

IMPORTANT: Filter out garbage entries before inserting:
  - Skip entries where city looks like a county name (e.g., "Miami-Dade County")
  - Skip entries that are clearly not city names (numbers, single chars, etc.)
  - Skip state-level entries (city = state name)
  - Use the CITY_REGISTRY and us_cities table as validation sources

Print summary: "Added X cities to prod_cities. Total now: Y"
