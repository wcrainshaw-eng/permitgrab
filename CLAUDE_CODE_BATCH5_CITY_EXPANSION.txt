This batch expands PermitGrab from its current ~11 cities to 30+ cities by tapping into free public open data APIs across three platforms: Socrata (SODA API), ArcGIS Open Data Hub, and CKAN. NO scraping required — these are all structured JSON APIs.

IMPORTANT: The existing permit collector pattern should be refactored into a modular system where each city is a configuration entry, not a hardcoded function. This makes adding future cities trivial.

=============================================
PART 1: REFACTOR COLLECTOR INTO MODULAR SYSTEM
=============================================

Refactor the permit collection system so each city is defined by a config entry rather than custom code. Create a file called city_configs.py with a CITY_REGISTRY dictionary:

Each city config entry should contain:
{
    "name": "Nashville",
    "state": "TN",
    "slug": "nashville",
    "platform": "socrata" | "arcgis" | "ckan" | "carto",
    "base_url": "https://data.nashville.gov",
    "dataset_id": "3h5w-q8b7",
    "field_map": {
        "permit_number": "permit_number",
        "address": "address",
        "description": "const_cost",
        "permit_type": "permit_type_mapped",
        "value": "const_cost",
        "date_issued": "date_issued",
        "status": "status",
        "contractor": "contractor",
        "owner": "owner"
    },
    "default_filter": "$where=date_issued > '{ninety_days_ago}'",
    "active": true
}

Create platform-specific fetcher functions:
- fetch_socrata(config) — uses SODA API: {base_url}/resource/{dataset_id}.json?{filter}&$limit=5000
- fetch_arcgis(config) — uses ArcGIS REST: {base_url}/api/download/v1/items/{dataset_id}/csv (or /query endpoint with f=json)
- fetch_ckan(config) — uses CKAN API: {base_url}/api/3/action/datastore_search?resource_id={dataset_id}&limit=5000
- fetch_carto(config) — uses CARTO SQL API: {base_url}/api/v2/sql?q=SELECT * FROM {table} WHERE {filter}

Each fetcher normalizes the response into the standard permit format used by the rest of the app.

Migrate ALL existing cities (NYC, Chicago, LA, Austin, Denver, Seattle, Portland, SF, and any others currently live) into this config system first, verify they still work, then add the new cities below.
