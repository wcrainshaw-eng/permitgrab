## V526 shipped (commit c3b37eb)

**Diagnosis pivot:** original hypothesis was Phase 3 memory bail. Actual
root cause is more interesting — the V488 stale-first sort ranked
cities by `MAX(permits.collected_at)`, which only advances when the
source returns new permits. Idle-source cities (visited every cycle
but no new permits in weeks) kept "ageing" under that metric and
dominated the budget. Production proof: gainesville-fl ran 51×/24h,
cary/clarksville/fulton/granger 36×/24h each; meanwhile 1,319 cities
never visited in 21+ days.

**Fix:** sort by `MAX(scraper_runs.run_started_at)` instead — that
column advances on every visit attempt regardless of source output.
True round-robin. NULL-last-run cities go first to bootstrap the
rotation. Math: 75 × 6 cycles/day = 450/day → ~1,250 typed cities
fully rotate in 2.8 days.

4 regression tests pin the new behavior, including the EXACT production
failure shape (visit-frequent-source-idle vs visit-stale-permits-fresh).
File-level guard catches future drift back to MAX(collected_at).
